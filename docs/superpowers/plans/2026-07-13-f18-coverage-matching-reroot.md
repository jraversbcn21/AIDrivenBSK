# F18 — Re-root the map to restore coverage matching — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Make `pnpm plan`'s evidence→map coverage matching report real coverage again by re-rooting the crawl on the `/es/` locale entry (dropping the bare `/` seed) and fixing the redirect-induced chain truncation (audit F4) that dropping `/` would otherwise unmask.

**Architecture:** Two one-line code changes in `explorer/` — remove `'/'` from the seed list, and make a discovered child's `discoveredVia` the parent's *resolved* path (`extraction.meta.path`) instead of the *requested* path (`item.path`). The planner's matching logic is untouched. The committed canonical map is then regenerated by a full live re-crawl and re-validated end-to-end.

**Tech Stack:** TypeScript, Playwright (crawl + e2e), Vitest (unit), pnpm, tsx CLIs (`explorer/cli.ts`, `planner/cli.ts`, `builder/cli.ts`). Runs against DES pre-prod over VPN.

**Model routing:** This is implementation work → **Claude Sonnet 5** (CLAUDE.md "Model routing policy"). The design/spec phase (Opus 4.8) is complete: `docs/superpowers/specs/2026-07-13-f18-coverage-matching-reroot-design.md`.

### Task 1: Drop `/` from seeds + fix F4 chain truncation (code)

The seed removal is harmful without the F4 fix (it unmasks chain truncation), and the F4 fix is inert without the seed removal (for non-redirecting parents, requested === resolved). They must land together and are reviewed as one unit. `crawlSession` has no unit-test harness (it needs a real `BrowserContext`), so the deterministic guarantee lands as a **contract test at the `buildMap` boundary**; the crawler line itself is proven by Task 2's live re-crawl (project precedent: M2/M8/M8b crawler changes are live-validated).

**Files:**
- Modify: `explorer/cli.ts:18` (the `SEEDS` constant)
- Modify: `explorer/crawl/crawler.ts:110-117` (child `discoveredVia`)
- Test: `explorer/map/builder.unit.test.ts` (add two contract tests)

**Interfaces:**
- Consumes: `buildMap({ classified, environment })` and the `page(path, discoveredVia)` test helper already defined in `explorer/map/builder.unit.test.ts`.
- Produces: no new exported symbols. Behavioral contract asserted: a child whose `discoveredVia` equals its parent's *resolved* `meta.path` reconstructs the full root→leaf chain; a child referencing the parent's pre-redirect *requested* path truncates.

- [ ] **Step 1: Write the two contract tests**

Add these two `it(...)` blocks inside the existing `describe('buildMap', ...)` in `explorer/map/builder.unit.test.ts` (the `page(path, discoveredVia)` helper already exists at the top of the file):

```ts
  it('reconstructs the full chain when a redirecting seed root is recorded under its resolved path (F4/F18)', () => {
    // Post-fix crawler output: the `/es/` seed server-resolves (gender gate) to
    // `/es/h-woman.html`, so the page is recorded under its RESOLVED path with
    // discoveredVia 'seed', and its child carries that resolved path as discoveredVia
    // (the F4 fix). The chain must reconstruct root->leaf, not truncate.
    const m = buildMap({
      classified: [
        { extraction: page('/es/h-woman.html', 'seed'), classification: { pageType: 'Home', confidence: 0.7 } },
        { extraction: page('/es/shop-cart.html', '/es/h-woman.html'), classification: { pageType: 'Cart', confidence: 0.8 } },
      ],
      environment: 'des',
    });
    const cartFlow = m.flows.find((f) => f.type === 'Cart');
    expect(cartFlow?.steps).toEqual(m.pages.map((p) => p.id));
    expect(cartFlow?.name).toBe('/es/h-woman.html -> /es/shop-cart.html');
  });

  it('truncates the chain when a child references the requested (pre-redirect) seed path — the F4 bug this fixes', () => {
    // Pre-fix crawler output: children of the redirected `/es/` seed carried the
    // REQUESTED path `/es` as discoveredVia, but the parent is indexed under its
    // resolved `/es/h-woman.html`. The lookup misses and the chain truncates to one
    // step. Documents exactly why the crawler must emit the resolved parent path.
    const m = buildMap({
      classified: [
        { extraction: page('/es/h-woman.html', 'seed'), classification: { pageType: 'Home', confidence: 0.7 } },
        { extraction: page('/es/shop-cart.html', '/es'), classification: { pageType: 'Cart', confidence: 0.8 } },
      ],
      environment: 'des',
    });
    const cartFlow = m.flows.find((f) => f.type === 'Cart');
    expect(cartFlow?.steps).toHaveLength(1); // truncated — parent `/es` is not a page in the map
  });
```

Change `discoveredVia: item.path` to use the resolved path, with an explaining comment:

```ts
      for (const href of extraction.links) {
        if (!isSameOrigin(href, deps.baseURL)) continue;
        frontier.add({
          path: normalizePath(href, deps.baseURL),
          session,
          depth: item.depth + 1,
          // Resolved parent path, not the requested `item.path`: children are discovered on
          // the page as it actually resolved (`extraction.meta.path` = page.url()), and
          // buildMap indexes parents under their resolved meta.path. Using the requested path
          // truncates every chain rooted at a redirecting seed (audit F4, unmasked by F18's
          // drop of the non-redirecting `/` seed). Inert when requested === resolved.
          discoveredVia: extraction.meta.path,
        } satisfies FrontierItem);
      }
```
